# 웹 3쿠션 당구 물리 명세서 (MVP)

## 1. 문서 정보
- 제목: 웹 3쿠션 당구 물리 명세서
- 버전: v1.0
- 작성일: 2026-02-20
- 범위: 샷 초기 선속도, 초기 각속도(스핀), 미스큐 한계, 핵심 물리 상수

## 2. 목적
이 문서는 MVP에서 현실적인 3쿠션 플레이 감각을 만들기 위한 최소 물리 규칙을 정의한다.

## 3. 상수
### 3.1 공
- 지름: `d = 0.0615 m`
- 반지름: `R = 0.03075 m`
- 질량: `m_b = 0.21 kg`
- 재질: 페놀 수지
- 공-공 반발계수: `e_bb = 0.92 ~ 0.98`

### 3.2 당구대
- 내경 플레이 영역: `2.844 m x 1.422 m`
- 외경: `3.100 m x 1.700 m`
- 쿠션 높이: `0.037 m`
- 공-쿠션 반발계수: `e_bc = 0.70 ~ 0.75`
- 미끄럼 마찰계수: `mu_s ~= 0.20`
- 구름 마찰계수: `mu_r ~= 0.01 ~ 0.015`

### 3.3 큐
- 큐 질량: `m_c = 0.50 kg`
- 큐 길이: `1.41 m` (충격량 계산에는 비핵심)
- 팁 지름: `12 mm`
- 팁-공 마찰계수: `mu_tip = 0.70`
- 팁 반발계수: `e_tip = 0.70` (기본값, 튜닝 가능)

## 4. 좌표계 및 입력 규약
- 수구 중심 타격점을 `(0, 0)`으로 둔다.
- 중심 기준 가로 오프셋은 `x`(좌우 회전)이다.
- 중심 기준 세로 오프셋은 `y`(상하 회전)이다.
- 샷 방향 단위 벡터를 테이블 평면상의 `n_hat`으로 둔다.
- 큐 수평 회전(좌우 마우스):
  - 마우스를 좌측으로 이동하면 큐가 시계방향으로 회전한다.
  - 마우스를 우측으로 이동하면 큐가 시계 반대방향으로 회전한다.
  - 수평 회전 범위는 `0 ~ 360도`이며 래핑 처리한다.
- 큐 고각 조절(상하 마우스):
  - 마우스를 위로 올리면 큐 고각이 증가하고 수구 상단을 겨냥한다.
  - 마우스를 아래로 내리면 큐 고각이 감소하고 수구 하단을 겨냥한다.
  - 고각 범위는 `0 ~ 89도`로 클램프한다.
- 스트로크 입력:
  - 좌클릭 유지 -> 아래로 드래그 -> 좌클릭 해제로 샷한다.
  - 유효 드래그 범위는 `10 px ~ 400 px`이며 클램프한다.
- 스핀 입력:
  - `W`: 12시 방향
  - `S`: 6시 방향
  - `A`: 9시 방향
  - `D`: 3시 방향

## 5. 드래그-속도 매핑 (GDD 정합)
드래그 거리를 `d_px`로 둔다.

- `d_px_clamped = clamp(d_px, 10, 400)`
- 최소 초기 공 속도: `V0_min = 1.0 m/s`
- 최대 초기 공 속도: `V0_max = 50 km/h = 13.89 m/s`
- 선형 매핑:
`V0_target = V0_min + (d_px_clamped - 10) / (400 - 10) * (V0_max - V0_min)`

`V0_target`은 UI 입력 규격에서 정의한 목표 초기 공 속도다.

## 6. 타격 직후 초기 선속도
큐와 수구의 물리 충돌식을 적용한다.

`V0 = (m_c * (1 + e_tip) / (m_c + m_b)) * v_c`

- `V0`는 물리식으로 계산된 타격 직후 수구 속력이다.
- 초기 선속도 벡터:
`v0_vec = V0 * n_hat`

구현 메모:
- 조작 규격과 물리식을 일치시키기 위해 `V0_target`에서 `v_c`를 역산한다:
`v_c = V0_target * (m_c + m_b) / (m_c * (1 + e_tip))`
- 이후 물리식으로 `V0`를 재계산하고 최종 안전 클램프를 적용한다:
`V0 = clamp(V0, V0_min, V0_max)`

## 7. 초기 각속도(스핀)
참고식(충격량 + 강체 관성모멘트 결합)을 사용한다.

- 구의 관성모멘트: `I = (2/5) * m_b * R^2`
- 상하 스핀 성분:
`omega_x = (5 * V0 * y) / (2 * R^2)`
- 좌우 스핀 성분:
`omega_z = (5 * V0 * x) / (2 * R^2)`

주의:
- 축 이름은 엔진 좌표계에 따라 달라질 수 있다. 일부 엔진에서는 좌우 스핀이 `omega_y`일 수 있다.
- 부호(`+/-`)는 프로젝트의 우수법칙 및 카메라/테이블 축 정의를 기준으로 통일한다.

## 8. 미스큐 규칙 (필수 제약)
당점이 공 가장자리에 너무 가까우면 미스큐로 처리한다.

- 오프셋 거리:
`r_off = sqrt(x^2 + y^2)`
- 미스큐 임계치:
`r_off > 0.9 * R`

미스큐 발생 시:
1. 샷 실패로 판정한다.
2. 수구에 유효 충격량을 적용하지 않는다(또는 큰 파워 손실을 적용한다).
3. 턴 종료 처리는 게임 규칙 로직을 따른다.

## 9. 샷 처리 순서 (런타임)
1. 방/경기/턴 상태를 검증한다.
2. 샷 입력(방향, 드래그 거리 `d_px`, 오프셋 `(x, y)`)을 읽는다.
3. 미스큐 검사(`r_off > 0.9R`)를 수행한다.
4. 유효 샷이면 `d_px_clamped`와 `V0_target`을 계산한다.
5. `V0_target`에서 `v_c`를 역산하고 최종 `V0`를 계산/클램프한다.
6. 타격 프레임에서 `Rigidbody.velocity = v0_vec`를 1회 세팅한다.
7. `omega_x`, `omega_z`를 계산해 `Rigidbody.angularVelocity`를 1회 세팅한다.
8. 이후 공-공/공-쿠션/마찰은 엔진 물리로 시간 전개한다.

## 10. 캘리브레이션 가이드
- 시작값 권장:
  - `e_tip = 0.70`
  - `e_bb = 0.95`
  - `e_bc = 0.72`
  - `mu_s = 0.20`
  - `mu_r = 0.012`
- 튜닝 순서:
1. 샷 진행 거리(파워 매핑과 `e_tip`)
2. 공-공 반발 감각(`e_bb`)
3. 쿠션 반발 각도/에너지(`e_bc`)
4. 미끄럼->구름 전이와 감속 꼬리(`mu_s`, `mu_r`)

## 11. 검증 체크리스트
- 중앙 타격(`x=0, y=0`)에서 스핀이 거의 0에 가깝다.
- `|y|`가 커질수록 상하 스핀이 증가한다.
- `|x|`가 커질수록 좌우 스핀이 증가하고 쿠션 이후 궤적 변화가 커진다.
- `r_off > 0.9R`에서 미스큐가 일관되게 발생한다.
- 같은 입력/틱레이트에서 수구 결과가 결정론적으로 재현된다.
- 드래그 `10 px`는 `1.0 m/s`, `400 px`는 `13.89 m/s`가 일관되게 나온다.
